# 程式碼審查
 
1. 讀取變更：`git diff --stat main...HEAD` 和 `git diff main...HEAD`
2. 審查清單：
   **安全**：無硬編碼密鑰、SQL 參數化、輸入驗證、無 console.log
   **效能**：無 N+1 查詢、大量資料分頁、適當用 Promise.all
   **可維護**：函式 <50 行、錯誤處理、命名有意義
   **測試**：新功能有測試、邊界條件覆蓋
3. 產出審查報告，列出問題和建議（繁體中文）
